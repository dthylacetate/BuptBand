# 项目需求规格说明书- “拼好队” 

**版本：** v1.0

**目标：** 构建一个针对校园/业余乐手的垂直招募与技术交流平台。

------

## 1. 项目概述

### 1.1 背景与目标

针对北邮校内音乐爱好者在寻找队友、展示个人才华及器材讨论时缺乏垂直平台的痛点，本项目旨在通过 Web 端提供：

- 精准的双向匹配招募系统。
- 乐手可视化名片。
- 高质量技术讨论社区。

### 1.2 目标用户

- **个人乐手：** 寻找乐队或志同道合的排练伙伴。
- **乐队/组织：** 处于初创或运营中，需要补强特定位置。
- **技术爱好者：** 关注乐理、扒谱及闲聊

------

## 2. 用户角色定义 

| **角色**            | **描述**   | **核心权限**                                      |
| ------------------- | ---------- | ------------------------------------------------- |
| **未登录访客**      | 浏览者     | 查看招募列表、乐手公开名片、阅读讨论帖。          |
| **认证用户 (乐手)** | 核心参与者 | 发布/编辑个人名片、发布求带信息、在讨论区回帖。   |
| **乐队/项目创建者** | 招募发起人 | 创建并管理乐队/项目主页，发布具体位置的招募信息。 |

------

## 3. 功能需求 

### 3.1 招募板块 

该板块采用**双向分类**逻辑：

#### A. 乐队/项目招募 

- **发布功能：** 用户可创建项目并勾选“招募位置”（多选，如贝斯、键盘）。
- **必填字段：** 项目名称、风格、技术要求简述、排练频次。
- **列表展示：** 默认按时间倒序排列，支持根据“位置”和“性质”进行筛选。

#### B. 乐手求带 

- **发布功能：** 个人发布求带意愿。
- **核心字段：** 主打乐器、意向风格、空档时间。
- **展示逻辑：** 突出乐手的主打乐器标签，方便乐队快速定位。

### 3.2 乐手可视化名片 

- **基础信息：** 昵称、校区、本命乐器、琴龄（10级/年限等）。
- **硬核装备库 (Gear Hub)：**
  - **文字清单：** 用户可填写的设备清单。
  - **图片集：** 支持上传 3-5 张图片。**MVP 要求：** 前端图片压缩，后端防盗链处理。
- **多媒体：** 支持填写 Bilibili 或 YouTube 视频 URL，前端实现 iframe 嵌入播放。

### 3.3 讨论区 

- **分类体系：** 预设“技术/乐理”、“闲聊树洞”两个板块。
- **发布与互动：** 支持富文本（或简单图文）发帖、评论回复。
- **排序：** MVP 版本仅支持按“最新发布时间”排序。

------

## 4. 非功能性需求 

### 4.1 性能与体验

- **响应式布局：** 适配 PC 与移动端浏览器。
- **加载速度：** 首屏渲染时间（LCP）应控制在 2s 内。
- **图片优化：** 上传的装备图自动生成缩略图，减少流量消耗。

### 4.2 安全性

- **数据隔离：** 用户的联系方式（如微信/手机）默认隐藏，仅对登录用户展示。
- **内容审核：** 预留简单的关键词过滤接口，防止违规内容发布。

------

## 5. 验收标准 

1. 用户能成功注册并填写一份可以包含图片的乐手名片。
2. 乐队发起者能发布一条“寻贝斯手”的招募，且在列表页正确显示。
3. 用户在讨论区发帖后，首页能实时更新该帖。
4. 视频链接填写后，在名片页能正常加载播放器。
